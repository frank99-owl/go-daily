# 기술 아키텍처 및 핵심 모듈 (ARCHITECTURE)

이 문서는 go-daily의 내부 구조, `lib/` 디렉토리의 "6개 도메인 분리" 리팩토링 및 중앙 집중식 미들웨어 로직에 대해 설명합니다.

## 1. 글로벌 요청 생명주기 (`proxy.ts`)
모든 사용자 대상 요청은 `proxy.ts` 미들웨어를 통과합니다. 이 미들웨어는 단일 패스로 다음과 같은 세 가지 핵심 작업을 처리합니다:
1.  **세션 갱신 (Auth Refresh)**: `@supabase/ssr`을 사용하여 페이지 이동 시마다 세션 쿠키를 갱신함으로써, 서버 컴포넌트(RSC)가 항상 최신 사용자 상태를 유지하도록 합니다.
2.  **로케일 협상 (Locale Negotiation)**: 308(영구) 리다이렉트 매트릭스를 처리하여 모든 경로에 언어 접두사(`/{zh|en|ja|ko}/...`)가 포함되도록 보장합니다.
3.  **경로 보호 (Route Guarding)**: 보호된 경로(예: `/account`, `/pricing/checkout`)를 가로채어 인증되지 않은 사용자를 로그인 페이지로 유도합니다.

## 2. 핵심 도메인 모듈 (`lib/`)

### `lib/auth/` & `lib/supabase/`
*   **이중 클라이언트 전략**: 브라우저 환경을 위한 `client.ts`와 App Router의 비동기 서버 컴포넌트를 위한 `server.ts`를 별도로 운용합니다.
*   **권한 서비스 계층**: `service.ts`는 `service_role` 키를 사용하여 RLS(행 수준 보안)를 우회하며, Stripe 웹훅이나 Cron 이메일 등 백그라운드 작업을 처리합니다.

### `lib/storage/` (영속성 엔진)
시스템은 계층화된 동기화 모델을 채택하고 있습니다:
1.  **익명 계층 (LocalStorage)**: 비로그인 사용자를 위한 1차 저장소입니다.
2.  **큐 계층 (IndexedDB)**: 대기 중인 연습 기록은 영구적인 IndexedDB 큐에 저장됩니다. 이를 통해 동기화 완료 전에 탭을 닫아도 데이터가 손실되지 않습니다.
3.  **클라우드 계층 (Supabase)**: 최종적인 데이터 진실 공급원(Source of Truth)입니다. `syncStorage.ts`는 `online` 이벤트나 페이지 로드에 의해 트리거되는 "플러시(Flush)" 로직을 오케스트레이션합니다.

### `lib/coach/` (AI 지능)
*   **프롬프트 관리**: `coachPrompt.ts`에 집중화하여 서로 다른 문제 사이에서도 "소크라테스식 교육" 스타일이 일관되게 유지되도록 합니다.
*   **예산 제어**: `coachBudget.ts`는 DeepSeek 과금 상위 단계에서 애플리케이션 레벨의 엄격한 월간 토큰 제한을 적용하여 예상치 못한 비용 급증을 방지합니다.

### `lib/i18n/` (글로벌 존재감)
*   **경로 우선**: 검색 엔진이 4,800개 이상의 로컬라이즈된 페이지를 독립적으로 크롤링할 수 있도록 쿠키가 아닌 URL 파라미터를 통한 언어 상태 유지를 선호합니다.
*   **정확성 검증**: `validateMessages.ts`를 통해 `zh`, `en`, `ja`, `ko` 사이의 번역 키가 빌드 시점에 항상 동기화되도록 보장합니다.

## 3. 데이터 흐름: 연습 기록의 생명주기
1.  **이벤트**: 사용자가 바둑판(`GoBoard.tsx`)에서 수를 두고 문제를 해결합니다.
2.  **로컬 쓰기**: `saveAttempt`가 LocalStorage에 즉시 기록하여 빠른 피드백을 제공합니다.
3.  **큐 추가**: 로그인 상태인 경우 연습 기록이 IndexedDB 큐에 추가됩니다.
4.  **동기화**: `syncStorage`가 Supabase의 `attempts` 테이블에 일괄 삽입(Batch Insert)을 시도합니다.
5.  **권한 갱신**: 동기화 성공 후 사용자의 연승 기록(Streak) 및 SRS 일정 재계산이 트리거됩니다.

## 4. 보안 및 규제 준수
*   **행 수준 보안 (RLS)**: 모든 Postgres 테이블에서 `auth.uid() = user_id` 정책을 강제하여 데이터베이스 계층에서의 데이터 유출을 원천 차단합니다.
*   **PII 마스킹**: Sentry와 PostHog는 `beforeSend` 필터로 구성되어 있으며, AI 코치와의 대화 내용이 클라이언트를 떠나기 전에 개인정보를 비식별화합니다.
