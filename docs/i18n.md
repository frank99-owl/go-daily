## 1. 总览

go-daily 支持 **4 种语言**：

| 代码 | 语言 | 文件 |
|---|---|---|
| `zh` | 中文（简体） | `content/messages/zh.json` |
| `en` | English | `content/messages/en.json` |
| `ja` | 日本語 | `content/messages/ja.json` |
| `ko` | 한국어 | `content/messages/ko.json` |

**默认语言**：`zh`（中文）  
**持久化**：用户选择的语言存储在 `localStorage["go-daily.locale"]`，下次访问自动恢复。

---

## 2. 目录结构

```
content/messages/
  en.json   ← 类型主文件（所有语言以此为准）
  zh.json
  ja.json
  ko.json

lib/
  i18n.tsx       ← LocaleProvider + useLocale hook
  localized.ts   ← LocalizedText 的回退辅助函数
```

## 3. 运行机制

### 3.1 类型安全

`lib/i18n.tsx` 的核心设计：

```ts
import en from "@/content/messages/en.json";

// Messages 类型从 en.json 自动推导——无需手写类型定义
type Messages = typeof en;

const DICTS: Record<Locale, Messages> = { zh, en, ja, ko };
```

**关键点**：`type Messages = typeof en` 意味着 `en.json` 是唯一的"类型权威"。  
当你向 `en.json` 添加新 key 时，TypeScript 会立即要求 `zh.json`、`ja.json`、`ko.json` 也必须包含同名 key，否则编译报错。

### 3.2 Context + Hook

```ts
type LocaleContextValue = {
  locale: Locale;       // 当前语言代码
  setLocale: (l: Locale) => void;  // 切换语言
  t: Messages;          // 当前语言的翻译对象
};
```

`LocaleProvider` 在 `app/layout.tsx` 包裹整个应用，保证所有子组件可通过 `useLocale()` 访问 `locale`、`setLocale` 和 `t`。

### 3.3 语言切换流程

```
用户点击语言切换按钮
  → setLocale("en")
    → setLocaleState("en")       // React state 更新，触发重渲染
    → localStorage.setItem(...)  // 持久化，下次访问恢复
```

## 4. 在组件中使用翻译

### 4.1 基本用法（客户端组件）

```tsx
"use client";
import { useLocale } from "@/lib/i18n";

export function MyComponent() {
  const { t, locale, setLocale } = useLocale();

  return (
    <div>
      <h1>{t.home.todaysProblem}</h1>
      <p>{t.result.correct}</p>
    </div>
  );
}
```

### 4.2 带参数的翻译（插值）

当前 `t.*` 中部分字符串包含 `{{placeholder}}`，需要手动替换：

```ts
// en.json: "tally": "{{solved}} solved · {{attempted}} attempted · {{unattempted}} todo"
const tally = t.puzzles.tally
  .replace("{{solved}}", String(stats.solved))
  .replace("{{attempted}}", String(stats.attempted))
  .replace("{{unattempted}}", String(stats.unattempted));
```

> 目前没有第三方 i18n 库（如 `next-intl`）；插值完全手动。若插值需求增多，可考虑引入 `next-intl`（见[常见问题](#8-常见问题)）。

### 4.3 服务端组件 / RSC

服务端组件不能使用 `useLocale()`（不能用 hooks）。  
服务端目前无需 i18n——所有需要翻译的 UI 均在客户端组件（`"use client"`）中渲染。  
若未来服务端需要翻译，可直接 import 对应语言 JSON：

```ts
// 服务端组件（仅示例，目前项目未使用）
import zh from "@/content/messages/zh.json";
const t = zh;
```

## 5. LocalizedText 与 `localized()` 辅助函数

### 5.1 LocalizedText

`Puzzle` 对象中，`prompt` 和 `solutionNote` 是 `LocalizedText` 类型：

```ts
type LocalizedText = Record<Locale, string>;
// 即 { zh: string; en: string; ja: string; ko: string }
```

### 5.2 `localized()` 函数

源文件：`lib/localized.ts`

```ts
import type { Locale, LocalizedText } from "@/types";
const FALLBACK_ORDER: Locale[] = ["en", "zh", "ja", "ko"];

export function localized(text: LocalizedText, locale: Locale): string {
  const v = text[locale];
  if (v) return v;
  // 当前语言缺失时按回退顺序查找
  for (const fb of FALLBACK_ORDER) {
    if (fb === locale) continue;
    const alt = text[fb];
    if (alt) return alt;
  }
  return "";
}
```

**使用场景**：题目文本渲染、AI 教练系统 prompt、搜索过滤。

**不要**直接写 `puzzle.prompt[locale]`——这在某语言字段为空时会渲染 `undefined`。

**统一写法**：

```tsx
// ✅ 正确
import { localized } from "@/lib/localized";
<p>{localized(puzzle.prompt, locale)}</p>

// ❌ 错误（可能渲染 "undefined"）
<p>{puzzle.prompt[locale]}</p>
```

## 6. 新增翻译 key

**场景**：你在 UI 中需要一段新的文案，如 `t.home.newFeature`。

1. **在 `en.json` 中添加（唯一权威）**：
   ```json
   "home": {
     "todaysProblem": "Today's Problem",
     "newFeature": "New Feature"   // ← 添加这里
   }
   ```

2. **TypeScript 编译器会报错**，提示 `zh.json`、`ja.json`、`ko.json` 缺少对应 key：
   ```
   Type '{ todaysProblem: string; }' is missing the following properties
   from type 'typeof en["home"]': newFeature
   ```

3. **在其他三个 JSON 中添加对应翻译**：
   ```json
   // zh.json
   "newFeature": "新功能"
   // ja.json
   "newFeature": "新機能"
   // ko.json
   "newFeature": "새 기능"
   ```

4. 在组件中使用 `{t.home.newFeature}`，IDE 自动补全。

> **无需重启开发服务器**——JSON 文件变化会被 Next.js 热更新自动识别。

## 7. 新增语言

以添加法语（`fr`）为例：

### 步骤 1：新建翻译文件

```bash
cp content/messages/en.json content/messages/fr.json
# 然后翻译 fr.json 中的所有值
```

### 步骤 2：更新 `types/index.ts`

```ts
// 旧
export type Locale = "zh" | "en" | "ja" | "ko";
// 新
export type Locale = "zh" | "en" | "ja" | "ko" | "fr";
```

### 步骤 3：更新 `lib/i18n.tsx`

```ts
import fr from "@/content/messages/fr.json";

const DICTS: Record<Locale, Messages> = { zh, en, ja, ko, fr };
```

### 步骤 4：更新回退链 `lib/localized.ts`

```ts
const FALLBACK_ORDER: Locale[] = ["en", "zh", "ja", "ko", "fr"];
```

### 步骤 5：更新语言切换 UI

在语言选择组件中添加 `fr` 选项，显示文案可用 `"Français"`。

### 步骤 6：更新验证脚本

`scripts/validatePuzzles.ts` 中有语言检查逻辑，需添加 `"fr"` 到相应的 locale 数组。

## 8. 常见问题

### Q: 为什么不用 `next-intl` 或 `react-i18next`？

当前规模（4 语言、约 100 个文案 key）不需要第三方库——Context + TypeScript 推导已足够类型安全且零依赖。  
若插值模板（`{{placeholder}}`）的使用大量增加，或需要复数规则、日期格式化，可引入 `next-intl`：迁移成本低，API 相似。

### Q: `useLocale()` 调用了但语言没切换？

检查组件是否在 `<LocaleProvider>` 内（`app/layout.tsx` 已包裹）。  
若在 RSC 中调用 `useLocale()` 会抛出错误：确保文件顶部有 `"use client"`。

### Q: 翻译 key 在某些语言中缺失怎么处理？

- `t.*`（UI 文案）：TypeScript 编译时强制所有语言文件结构一致，正常情况下不会缺失。
- `puzzle.prompt` / `puzzle.solutionNote`（题目数据）：使用 `localized()` 函数，它会自动回退到其他语言。

### Q: 构建时语言切换 hydration 会闪烁吗？

服务端渲染时语言为默认值 `zh`，客户端 mount 后从 localStorage 读取实际语言再更新——可能有一次短暂的 `zh → <user lang>` 闪烁。  
若要消除闪烁，可在 `app/layout.tsx` 中读取 cookie（需配合服务端 locale 检测）。
