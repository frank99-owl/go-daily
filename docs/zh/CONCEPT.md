# 项目理念与战略愿景 (CONCEPT)

本文件概述了 go-daily 从一个简单的 MVP 原型演变为全球化订阅产品的核心逻辑与战略设计。

## 1. 项目使命

Go-daily 旨在解决围棋练习中的“坚持难题”。我们将深奥、枯燥的棋类练习转化为一种基于 Next.js 与 AI 技术、每天仅需 5 分钟的现代化生活习惯。

## 2. 战略阶段 (增长逻辑)

### 第一阶段：地基构建与全球触达 (已完成)

将“玩具”打造成“产品”。

- **地理重心**：首发日韩（高付费意愿市场）并以英文版本兜底全球。
- **国际化路由**：从传统的 Cookie 模式转向 **URL 路径模式** (`/[locale]`)，以实现 4,800+ 个死活题页面的 SEO 最大化覆盖。
- **持久化方案**：实现了稳健的“三态同步”逻辑（匿名 LocalStorage -> IndexedDB 队列 -> Supabase 云端）。

### 第二阶段：商业化与 AI 教练 (启动中)

在“零成本”基础设施上验证商业模式。

- **Pro 会员包**：核心卖点定义为**跨设备同步**、**无限次 AI 教练**以及 **SRS 智能复习系统**。
- **定价策略**：通过 Stripe 实现针对日韩市场的动态定价，并强制执行 7 天试用期以提升转化。
- **AI 经济学**：利用 DeepSeek 实现专业级教学，成本仅为 KataGo 云端算力的零头。

### 第三阶段：合规化与内容深度 (未来)

实现项目的专业化与规模化。

- **合规性**：从占位符转向完整的 GDPR/JCT 合规，并实现统一的账号注销逻辑。
- **内容矩阵**：从 19x19 死活扩展至 9x9/13x13 入门路径，并补充布局与官子专题。

## 3. “精益”工程哲学

- **成本控制**：目标是预收入阶段 $0/月固定支出。仅保留按量付费 (LLM) 和收入触发 (Vercel Pro) 的成本。
- **数据一致性**：使用 **Attempt Dedup Key** (`puzzleId-solvedAtMs`) 作为全球唯一的同步锚点，确保多设备间数据不冲突。

## 4. 内容伦理与版权

- **自审计策略**：维护一份 `knownPublicSources.ts` 白名单。
- **分级开放**：仅将经过验证的公有领域题目置于 Pro 功能（如 SRS）之后，未知来源题目保持免费并标注社区归属。
