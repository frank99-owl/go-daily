# 收入实验计划 (REVENUE_EXPERIMENTS)

**生成日期**: 2026-05-18
**原则**: 小额、可回滚、合规、低承诺；先验证付费意愿与学习价值，不制造不可履约的商业承诺。

## 实验总原则

- 不向用户承诺棋力提升结果，只承诺具体功能、内容或服务交付。
- 不收集敏感隐私；访谈和报名只收集必要联系方式、围棋水平、学习目标和同意记录。
- 所有外部动作（公开 waitlist、收款、发邮件、联系老师/机构）都需要 Frank 单独批准。
- 每个实验设停止条件；数据不足时不扩大投入。

## 实验一：Pro waitlist

| 项目     | 内容                                                                     |
| -------- | ------------------------------------------------------------------------ |
| 假设     | 有围棋自学用户愿意为跨设备同步、SRS 错题复习和高额 AI 教练配额留下邮箱。 |
| 形式     | 发布一个低承诺等待名单，展示 Pro 权益边界和预计试用方式。                |
| 成功指标 | 50 个合格报名；报名页转化率 >= 8%；至少 10 人愿意接受 15 分钟访谈。      |
| 成本     | 文案、等待名单页面/表单、邮件确认；不需要立即开通付费。                  |
| 风险     | 邮箱合规、过度承诺、吸引非目标用户。                                     |
| 风险控制 | 明示未正式上线；不写“无限 AI”；提供删除报名入口；不发送营销轰炸。        |
| 停止条件 | 2 周内少于 15 个合格报名，或访谈显示 Pro 权益不是真实购买动机。          |

## 实验二：早鸟年费

| 项目     | 内容                                                                             |
| -------- | -------------------------------------------------------------------------------- |
| 假设     | 高频自学用户愿意用折扣年费换取 Pro 早期访问，但需要明确可退款边界。              |
| 形式     | 小规模邀请制，限定名额，Stripe live 仅在发布窗口批准后开启。                     |
| 成功指标 | 10 个付费用户或 3%+ waitlist 转付费；退款率 <= 10%；付费后 7 日内至少 3 次练习。 |
| 成本     | Stripe 配置、客服、退款处理、权益支持。                                          |
| 风险     | 税务/退款/试用规则、产品稳定性、AI 成本超预期。                                  |
| 风险控制 | 明确试用和退款政策；保留商业开关；设置 Coach 月度 token 预算。                   |
| 停止条件 | 退款/投诉集中指向价值不清，或 AI 成本超过收入的 35%。                            |

## 实验三：主题内容包

| 项目     | 内容                                                                           |
| -------- | ------------------------------------------------------------------------------ |
| 假设     | 用户愿意为更结构化的专题训练付费，而不只是为“题库数量”付费。                   |
| 形式     | 先做小型内容包，如“9x9 入门死活”“官子基础 20 题”“重复形变化专题”。             |
| 成功指标 | 30% 以上受访用户选择具体主题；首包完成率 >= 40%；至少 5 人愿意付费购买或预订。 |
| 成本     | 内容编辑、题目校对、解释与 wrongBranches 补齐。                                |
| 风险     | 内容版权、质量不稳定、专题范围过大。                                           |
| 风险控制 | 只使用可审来源；每批 20-50 题；保留人工审核清单。                              |
| 停止条件 | 完成率低于普通每日题，或用户无法说清内容包相对免费题的价值。                   |

## 实验四：付费用户访谈

| 项目     | 内容                                                                    |
| -------- | ----------------------------------------------------------------------- |
| 假设     | 目标用户愿意用较低补偿参与深度访谈，帮助验证学习闭环与付费触点。        |
| 形式     | 15-30 分钟远程访谈，可提供小额礼品卡或 Pro 试用码；不录制敏感信息。     |
| 成功指标 | 12 次有效访谈；至少 4 个重复出现的付费触发点；至少 3 个可执行产品改动。 |
| 成本     | 招募、补偿、访谈整理。                                                  |
| 风险     | 补偿吸引非目标用户、隐私记录过多。                                      |
| 风险控制 | 只记录产品相关摘要；不收集真实姓名以外的敏感信息；参与前告知用途。      |
| 停止条件 | 连续 5 次访谈无法产生新的学习痛点或付费假设。                           |

## 实验五：老师/机构合作试点

| 项目     | 内容                                                                                 |
| -------- | ------------------------------------------------------------------------------------ |
| 假设     | 围棋老师或小机构需要课后练习、错题复习和学习数据摘要，但会更重视可控内容与学生隐私。 |
| 形式     | 选择 1-2 位老师做非公开试点，提供班级练习包或复习建议摘要。                          |
| 成功指标 | 老师愿意每周布置一次练习；学生 7 日留存优于普通用户；老师愿意提供付费或转介绍意向。  |
| 成本     | 内容包准备、老师反馈沟通、可能的简单运营报表。                                       |
| 风险     | 未成年人隐私、机构采购周期长、定制需求失控。                                         |
| 风险控制 | 不收集学生敏感信息；先用匿名码或自愿账号；不承诺班级管理系统。                       |
| 停止条件 | 老师主要需求变成重定制 LMS，或试点无法在两周内形成可复用流程。                       |

## 推荐首轮顺序

1. 先做 Pro waitlist 与用户访谈，验证目标人群和付费触发点。
2. 同步准备一个小型主题内容包，把内容质量工作转成可展示价值。
3. waitlist 有明确需求后，再申请 Frank 批准 Stripe live 早鸟实验。
4. 老师/机构合作只做 1-2 个强关系试点，不做公开销售。
