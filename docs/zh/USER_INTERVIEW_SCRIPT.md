# 用户访谈脚本 (USER_INTERVIEW_SCRIPT)

**生成日期**: 2026-05-18
**目标**: 验证 go-daily 的学习价值、AI 信任边界、复习习惯和付费意愿。

访谈默认 20-30 分钟。不要收集身份证件、收入、家庭、未成年人详细信息、真实棋谱隐私或账号密码。若需要录音、发送邮件或联系外部用户，必须先获得 Frank 单独批准。

## 开场说明

- 这是产品研究访谈，不是棋力测试。
- 你可以跳过任何问题，也可以随时结束访谈。
- 我们只记录与产品体验相关的摘要，不记录敏感个人信息。
- 如果体验 AI 教练，AI 只能作为辅助解释，不代表职业棋手判断。

## 分组与筛选

| 目标用户        | 筛选问题                               | 重点                           |
| --------------- | -------------------------------------- | ------------------------------ |
| 围棋初学者      | 学棋多久？最近一个月是否做过死活题？   | 入门阻力、坚持难点、解释理解   |
| 复盘型用户      | 是否会记录错题或复盘？用什么工具？     | 错题复习、变例理解、SRS 价值   |
| 老师/机构       | 是否给学生布置课后练习？如何检查？     | 内容可控性、学生隐私、运营报表 |
| AI 学习工具用户 | 是否用过 ChatGPT/AI 学棋或学其他技能？ | AI 信任、边界、付费心理        |

## 核心问题

### 学习痛点

1. 你现在通常怎么练围棋？频率是多少？
2. 最近一次做死活题卡住时，你最想知道什么？
3. 你停止练习的最常见原因是什么？
4. 你更喜欢每天短练，还是集中刷题？为什么？

### 错题复习

1. 你会不会回看做错的题？如果会，怎么回看？
2. 做错后，什么信息最能帮助你下次避免同样错误？
3. 你希望系统提醒你复习，还是自己主动找错题？
4. “今天该练什么”对你有没有价值？你希望它基于什么判断？

### AI 信任

1. 如果 AI 解释一道围棋题，你会怎样判断它可信不可信？
2. 你能接受 AI 只回答当前题目范围内的问题吗？
3. 你更需要简短提示、完整讲解，还是一步步追问？
4. 如果 AI 说“不确定”或拒绝回答超范围问题，你会觉得更可信还是更失望？

### 付费意愿

1. 哪些功能让你愿意注册账号？
2. 哪些功能可能让你愿意付费？
3. 你更愿意按月、按年，还是一次性购买内容包？
4. 对你来说，Pro 的核心价值更像是 AI 教练、错题复习、多设备同步，还是专题内容？
5. 什么情况会让你立即取消或不愿意付费？

## 老师/机构加问

1. 你如何判断学生是否真的完成课后练习？
2. 你是否需要可控题单，而不是系统随机推荐？
3. 学生数据哪些可以看，哪些不应该收集？
4. 如果提供班级练习包，你希望按学生数、班级、内容包还是时间收费？

## 产品演示任务

让用户边操作边说想法：

1. 从首页进入今日题。
2. 完成一题，查看结果页。
3. 查看错因或解析。
4. 打开 Coach，尝试一个建议问题。
5. 进入 review 或 stats，判断下一步是否清楚。

观察但不要引导：

- 用户是否理解当前轮到谁下。
- 用户是否知道下一步该做什么。
- 用户是否信任 AI 答案的边界。
- 用户是否能说出 Pro 的具体价值。

## 结束问题

1. 如果只能保留一个功能，你希望是什么？
2. 如果让你推荐给棋友，你会怎么描述它？
3. 你最担心这个产品哪里不可靠？
4. 你是否愿意参与下一轮试用或等待名单？如果愿意，只收集必要联系方式。
