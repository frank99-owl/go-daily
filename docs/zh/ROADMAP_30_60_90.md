# 30/60/90 天路线图 (ROADMAP_30_60_90)

**生成日期**: 2026-05-18
**范围**: Phase 3 首轮完成后的发布验证、留存验证和增长运营节奏。

本路线图不等同于自动执行计划。任何 push、部署、公开发布、真实邮件、生产支付、DNS/SMTP/Stripe live 操作都需要 Frank 单独批准。

## 30 天：内容质量与发布验证

目标：把“本地可运行”变成“可小范围发布且可解释边界”的状态。

| 主题           | 交付                                                                              | 验收                                      |
| -------------- | --------------------------------------------------------------------------------- | ----------------------------------------- |
| 发布材料冻结   | README、case study、launch checklist、收入实验、访谈脚本完成首轮                  | 文档可由 Frank 直接审阅，不依赖外部页面   |
| 本地发布预检   | 跑 `validate:messages`、`lint`、`tsc --noEmit`、必要时 `preflight:prod` / `build` | 无 fail；warn 有解释                      |
| 内容质量批次   | 继续补 20-50 道高价值题的 `solutionSequence` / `wrongBranches`                    | `queue:content` 显示新增 coach-ready 候选 |
| Pro 文案边界   | 公开文案只写可交付权益，不写保证提高、不写无限 AI                                 | README、pricing、法律页一致               |
| 小范围访谈准备 | 明确访谈对象、脚本、记录模板、隐私边界                                            | 不收集敏感隐私，联系动作待审批            |

30 天关键指标：

- 首题完成率。
- 结果页继续率。
- Coach 使用后的次日回访。
- review 完成率。
- waitlist 报名或访谈意向数。

## 60 天：留存与付费验证

目标：验证用户是否愿意重复练习，以及愿意为什么付费。

| 主题           | 交付                                         | 验收                                    |
| -------------- | -------------------------------------------- | --------------------------------------- |
| Waitlist 实验  | 小范围发布 Pro waitlist                      | 合格报名、访谈转化、反对理由可记录      |
| 访谈闭环       | 完成 12 次有效访谈                           | 至少 4 个重复出现的学习痛点或付费触发点 |
| SRS 留存优化   | 根据真实错题行为调整 review/stats 文案和入口 | 复习完成率和次日回访有改善              |
| 主题内容包原型 | 准备 1 个小型内容包                          | 用户能说出它相对免费每日题的价值        |
| 成本观察       | 观察 Coach token、失败率、配额耗尽分布       | AI 成本与用户价值可解释                 |

60 天关键指标：

- D1 / D7 回访。
- 每用户每周完成题数。
- review 到期题完成率。
- Coach 完成率与错误率。
- Waitlist 到访谈/付费意向转化。

## 90 天：增长与运营节奏

目标：形成可重复的内容、增长、运营和回滚节奏。

| 主题          | 交付                                        | 验收                       |
| ------------- | ------------------------------------------- | -------------------------- |
| 发布节奏      | 每 1-2 周一个小版本：内容、体验或运营材料   | 每次发布有验证命令和回滚点 |
| 收入实验决策  | 决定继续 Pro、内容包、老师试点中的哪条线    | 有成功指标和停止条件支持   |
| 老师/机构试点 | 仅在合适时做 1-2 个非公开试点               | 不变成重定制 LMS           |
| 内容运营系统  | 把重复题、官子/布局、9x9/13x13 入门纳入队列 | 题库结构偏差逐步下降       |
| 合规与支持    | 明确退款、取消、隐私请求、安全报告处理 SOP  | 外部承诺与实际处理能力匹配 |

90 天关键指标：

- 月活练习用户。
- 付费转化或强付费意向。
- 内容包完成率。
- 支持请求响应时间。
- 生产错误率与支付/邮件异常数。

## 不做清单

- 不把所有 3033 题都宣传为完整 AI 教练题。
- 不承诺“快速升段”“保证提升”“专业棋手级指导”。
- 不在没有 live smoke 和回滚预案时公开收费。
- 不为了增长收集敏感隐私或学生个人资料。
- 不在没有内容质量流程的情况下扩大题库数量。
