# 技術アーキテクチャと核心モジュール (ARCHITECTURE)

本書では、go-daily の内部構造、および `lib/` ディレクトリの「6ドメイン分離」リファクタリングと集中型ミドルウェアのロジックについて解説します。

## 1. グローバル・リクエスト・ライフサイクル (`proxy.ts`)
すべてのユーザー向けリクエストは `proxy.ts` ミドルウェアを通過します。これは一度のパスで以下の 3 つの重要なタスクを処理します：
1.  **セッションの更新 (Auth Refresh)**：`@supabase/ssr` を利用して、ナビゲーションごとにセッション Cookie を更新し、サーバーコンポーネント (RSC) が常に最新のユーザー状態を保持できるようにします。
2.  **ロケール交渉 (Locale Negotiation)**：308（永久リダイレクト）マトリックスを処理し、すべてのパスに言語プレフィックス (`/{zh|en|ja|ko}/...`) が付与されるようにします。
3.  **ルートガード (Route Guarding)**：保護されたパス（例：`/account`, `/pricing/checkout`）を傍受し、未認証ユーザーをログインページに誘導します。

## 2. コアドメイン・モジュール (`lib/`)

### `lib/auth/` & `lib/supabase/`
*   **デュアルクライアント戦略**：ブラウザ環境向けには `client.ts` を、App Router の非同期サーバーコンポーネント向けには `server.ts` を使用します。
*   **特権サービス層**：`service.ts` は `service_role` キーを使用して RLS（行レベルセキュリティ）をバイパスし、Stripe Webhook や Cron メールなどのバックグラウンドタスクを処理します。

### `lib/storage/` (永続化エンジン)
システムは階層化された同期モデルを採用しています：
1.  **匿名層 (LocalStorage)**：未ログインユーザーのプライマリソース。
2.  **キュー層 (IndexedDB)**：保留中の試行データは、永続的な IndexedDB キューに保存されます。これにより、同期完了前にタブを閉じてもデータが失われません。
3.  **クラウド層 (Supabase)**：最終的な真実のソース。`syncStorage.ts` は、`online` イベントやページロードによってトリガーされる「フラッシュ (Flush)」ロジックを管理します。

### `lib/coach/` (AI インテリジェンス)
*   **プロンプト管理**：`coachPrompt.ts` に集約し、異なる問題間でも「ソクラテス式指導」のスタイルが一貫するようにします。
*   **予算制御**：`coachBudget.ts` は、DeepSeek の課金上流でアプリケーションレベルのハードな月間トークン制限を適用し、予期せぬコスト増大を防止します。

### `lib/i18n/` (グローバル展開)
*   **URL 優先**：検索エンジンが 4,800 以上のローカライズされたページを個別にクロールできるように、Cookie ではなく URL パラメータによる言語状態の保持を推奨しています。
*   **不整合の検知**：`validateMessages.ts` により、`zh`, `en`, `ja`, `ko` 間の翻訳キーがビルド時に常に同期されていることを保証します。

## 3. データフロー：解答記録のライフサイクル
1.  **トリガー**：ユーザーが盤面 (`GoBoard.tsx`) で着手し、問題を解決します。
2.  **ローカル書き込み**：`saveAttempt` が LocalStorage に即座に書き込み、レスポンスを返します。
3.  **エンキュー**：ログイン済みの場合、試行データが IndexedDB キューに追加されます。
4.  **同期**：`syncStorage` が Supabase の `attempts` テーブルへのバッチ挿入を試行します。
5.  **権限の更新**：同期成功後、ユーザーの連勝記録 (Streak) と SRS スケジュールの再計算がトリガーされます。

## 4. セキュリティとコンプライアンス
*   **行レベルセキュリティ (RLS)**：すべての Postgres テーブルで `auth.uid() = user_id` ポリシーを強制し、データベース層でのデータ漏洩を防止します。
*   **PII マスキング**：Sentry と PostHog は `beforeSend` フィルタで構成されており、AI コーチとの対話がクライアントを離れる前に個人情報を匿名化します。
