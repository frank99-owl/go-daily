# プロジェクト理念と戦略的ビジョン (CONCEPT)

本書は、go-daily がゼロベースの MVP からグローバルなサブスクリプション製品へと進化するための核心的なロジックと戦略について説明します。

## 概要

- **製品面：** 毎日の詰碁練習、**4 言語** UX、オプションの Pro 同期と SRS、**DeepSeek 連携のストリーミング AI コーチ**（ペルソナ・クォータ・`coachPrompt.ts` で問題ごとの解説を起点としたプロンプト保護）。
- **技術基盤：** Next.js 16 App Router、RLS 付き Supabase、段階的クライアント永続化、Stripe — `lib/` 配下を **9 ドメイン** に分割（詳細は[アーキテクチャ](ARCHITECTURE.md)）。
- **本書の範囲：** 使命、フェーズ戦略、リーンなエンジニアリング制約、コンテンツ倫理。

## 1. プロジェクトの使命

go-daily は、囲碁学習における「継続の難しさ」を解決するために設計されました。Next.js と AI テクノロジーを活用し、奥深くもハードルの高い囲碁の練習を、毎日わずか 5 分で完結する現代的なライフスタイルへと変革します。

## 2. 戦略的フェーズ（成長のロジック）

### フェーズ 1：基盤構築とグローバル展開（完了）

「玩具」から「製品」への脱却。

- **地理的重点**：日本と韓国（高い ARPU 市場）を優先し、英語バージョンでグローバルをカバー。
- **国際化ルーティング**：従来の Cookie ベースから **URL プレフィックス方式** (`/[locale]`) に移行。`sitemap.xml` が列挙するロケール別 URL は現在 **12,000 本超**（静的ルート・リスト課題・各題面 × 4 言語）で、`content/data/puzzleIndex.json` のコーパスに比例して増える。
- **永続化ソリューション**：堅牢な「三段階同期」ロジック（匿名 LocalStorage -> IndexedDB キュー -> Supabase クラウド）を実装。

### フェーズ 2：収益化と AI コーチ（展開中）

「ゼロコスト」のインフラでビジネスモデルを検証。

- **Pro プランの定義**：**マルチデバイス同期**、**大容量の AI コーチ枠**（**1 日 50+／月 1,000+** を目安に記載 — 詳細は[プロダクト仕様](PRODUCT_SPECS.md)）、および **SRS 記憶システム**を中核価値として提供。
- **価格戦略**：Stripe による日本・韓国向けの動的価格設定を導入。摩擦を減らすために 7 日間の無料トライアルを必須化。
- **AI エコノミー**：DeepSeek を活用し、KataGo などのクラウド計算リソースの数分の一のコストでプロレベルの指導を実現。

### フェーズ 3：コンプライアンスとコンテンツの拡充（次フェーズ）

プロフェッショナル化とスケーリング。

- **法的遵守**：プレースホルダーから完全な GDPR/JCT（消費税）対応への移行。統合されたアカウント削除ロジックの実装。
- **コンテンツの多様化**：19路盤の死活だけでなく、初心者向けの 9路/13路盤や、布石・ヨセのカテゴリを拡充。

## 3. 「リーン」エンジニアリング哲学

- **コスト管理**：収益化前の段階では固定費 0 円/月を目標とします。従量課金（LLM）と収益連動型（Vercel Pro）のコストのみに抑えます。
- **データ整合性**：**Attempt Dedup Key** (`puzzleId-solvedAtMs`) をグローバルな同期アンカーとして使用し、クロスプラットフォームでのデータ競合を防止します。

## 4. コンテンツ倫理と著作権

- **自己監査戦略**：`knownPublicSources.ts` ホワイトリストを維持します。
- **段階的公開**：検証済みのパブリックドメインまたはクリエイティブ・コモンズの詰碁のみを Pro 機能（SRS など）の対象とし、出典不明のコンテンツはコミュニティの帰属を明記した上で無料公開を維持します。
