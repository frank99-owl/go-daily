# Code of Conduct

## Our Standards

go-daily should be a focused, respectful place to discuss the product, the codebase, and Go learning. Contributors and maintainers are expected to:

- Be respectful and constructive in issues, pull requests, and reviews.
- Keep technical feedback specific, actionable, and tied to the work being discussed.
- Avoid harassment, personal attacks, discriminatory language, and sustained off-topic disruption.
- Respect privacy. Do not publish private contact details, credentials, vulnerability details, or user data.

## Scope

This code applies to all project spaces, including GitHub issues, pull requests, discussions, documentation, and any private project communication used to coordinate contributions.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests; block accounts; or take other reasonable action when behavior violates this code.

If you need to report conduct concerns, contact the repository owner privately. Do not use the security disclosure channel for conduct reports unless the issue also involves a vulnerability or exposed user data.
